Frequently Asked Questions

1. the difference between var, let, and const
var, let, and const are used to declare variables in JavaScript. var is old and less safe, let can change, and const cannot change once set. I use let and const more.
2. the difference between map(), forEach(), and filter()
map(), forEach(), and filter() are used for arrays in JavaScript. map() creates a new array, forEach() loops without returning, and filter() creates a new array with matching items.
3. Explain arrow functions and how they are different from regular functions
Arrow functions are a shorter way to write functions in JavaScript. They don’t have their own this, making them different from regular functions. Use them for simple and cleaner code.
4. How JavaScript Promises work
JavaScript Promises handle async tasks. A Promise has three states: pending, resolved, or rejected. It runs a task and, when done, either resolves (success) or rejects (error). Use .then() and .catch().
5. How closures work in JavaScript
Closures in JavaScript happen when a function remembers variables from its parent scope, even after the parent function has finished running. This helps keep data private and makes functions more powerful.
